# [Refactored] 지능형 휴머노이드 자율주행 및 미션 제어 시스템

## 1. 개요

본 프로젝트는 **제 20회 임베디드 소프트웨어 경진대회(지능형 휴머노이드 부문)** 참가 당시 작성했던 절차 지향적 제어 코드를 객체 지향 프로그래밍(OOP) 기반으로 재설계한 완성본이다.
<br>
<br>
기존에 성공적으로 동작했던 '동적 경로 탐색 알고리즘'과 '상태 머신'의 뼈대는 유지하되, 라즈베리 파이 4 환경에서의 메모리 누수 방지, 영상 처리 프레임 속도(FPS) 극대화, 그리고 코드의 유지보수성을 높이는 데 집중하여 시스템을 최적화했다.

---

## 2. 주요 리팩토링 및 최적화 사항

### 2.1 비전 센서(카메라) 리소스 병목 해결 및 클래스화 (alpha.py)
- **문제점** : 기존 코드에서는 프레임을 검사할 때마다 `cv2.VidieoCapture(0)`를 호출하여 카메라를 열고 닫는 과정을 반복했다. 이는 임베디드 보드에 심각한 I/O 병목과 딜레이를 유발했다.
- **개선사항** : 비전 처리 로직을 `Vision` 클래스로 캡슐화했다. 인스턴스 생성 시 카메라 포트를 단 한 번만 초기화하여 리소스를 점유하도록 변경함으로써, 90 FPS의 고속 프레임 환경을 지연 없이 안정적으로 유지한다.

### 2.2 구조적 개선: 객체 지향 설계(OOP) 및 상태 관리 (main.py)
- **문제점** : 메인 제어 루프가 방대한 중첩 `if-elif-else`문과 전역 변수들로 구성되어 있어, 특정 예외 상황 발생 시 상태 추적 및 디버깅이 어려웠다.
- **개선사항** : `HumanoidController` 클래스를 도입하여 모든 로봇의 상태(`step`, `direction`, `cardinal_dir`)를 인스턴스 변수로 통합 관리한다. 미션 수행 로직을 `perform_stair_mission()`, `perform_rescue_mission()` 등의 독립된 메서드로 분리하여 메인 루프의 가독성과 응집도를 높였다.
 
### 2.3 보행 및 조향 알고리즘의 모듈화
- **문제점** : 직진 중 라인을 벗어났을 때 방향을 보정하는 로직이 각 `step`구간마다 중복으로 빡빡하게 코딩되어 있었다.
- **개선사항** : `walk_cycle(trace)`메서드를 작성하여 보행과 조향을 하나의 모듈로 통합했다. 한 사이클 안에서 상단 라인의 중앙값을 실시간으로 추적하여 미세 조향(`left_turn()`, `right_turn()`)을 수행한 뒤 직진 보행을 하도록 설계하여, 코드 중복을 제거하고 주행 안정성을 향상 시켰다.

---

## 3. 알고리즘 설게

### 3.1 동적 목적지 갱신 기반 자율 주행
- 로봇은 경기장 진입 시 T자 교차로에서 인식한 방위 표지판(N, E, S, W)과 화살표 방향을 기반으로 스스로 목적지의 절대 좌표를 설정한다.
- 코너를 지날 때마다 주행 벡터(+1, -1)를 가감하여 현재 위치를 갱신하며, 목적지 좌표와 일치할 때만 탈출 미션을 수행하도록 설계된 상태 머신을 적용했다.

### 3.2 영상 처리 (HSV & Contour)
- 실전 경기장의 조명 변화에 유연하게 대응하기 위해 RGB 대신 HSV 색 공간을 활용하여 객체(알파벳, 시민, 가이드라인)를 추적한다.
- 픽셀 스캔 및 프로파일 분석을 통해 딥러닝 모델 탑재 없이도 임베디드 환경에서 높은 정확도로 방위 글자와 형태를 인식한다.
